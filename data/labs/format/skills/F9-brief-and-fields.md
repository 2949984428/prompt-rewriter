## name: F9-brief-and-fields

label: Brief + Fields
target_model: gpt-image-2

# 任务

把用户的图像 query 改写成 **任务句 + 字段表** 的双段中文结构。整体写法借鉴 OpenAI Cookbook 实操(logo 原例)与 fal.ai 五槽框架,但**最终产物全部使用中文** —— 用来验证 gpt-image-2 在"显式 use case + 结构化字段"的中文上下文里的表现。

# 格式范式

## 段 1 · 任务句(开场一行,15-35 字)

固定句式:

```
为【对象/品牌】生成一张【产物类型】[,简短补充: 行业/用途]。
```

- **产物类型必须明示**:`logo` / `编辑摄影` / `电影感单图` / `产品摄影` / `UI 界面图` / `信息图` / `海报` / `包装设计` / `书籍封面` / `品牌标识` 等
- **品牌名 / 引号内容 / 数字 verbatim** —— 逐字保留 query 原文,**中日韩文不音译,英文专名不汉化**(例:Nike 不写"耐克";"Field & Flour" 不译)
- 段 1 不堆"高级""精致""惊艳"这种空形容词;形容词全留给段 2 的具体字段

## 段 2 · 字段描述(每行一字段,空字段整行跳过)

按以下顺序输出,字段名用中文,值也用中文:

```
风格/介质:<介质 + 风格定调,可含摄影师/设计师参考>
构图:<构图 + 视角 + 镜头焦段>
主体:<主体 + 关键外貌特征>
场景/背景:<环境 / 时间 / 背景>
光线/氛围:<光源性质 + 方向 + 色温 + 整体氛围>
色彩搭配:<颜色之间的功能关系,不是颜色清单>
材质/纹理:<表面质感>
文字(原文保留):"<query 中需要出现在画面里的文字,逐字>"
约束:<必须保留的约束>
避免:<反例 / cliché 列表>
```

# 工种开关(决定哪些字段必填)


| 工种                   | 命中 use case               | 段 2 行数 | 必填差异                                      |
| -------------------- | ------------------------- | ------ | ----------------------------------------- |
| **细节累加(additive)**   | 编辑摄影 / 产品图 / 插画故事 / 电影感单图 | 6-10 行 | 构图 + 场景 + 光线 必填,避免 ≥ 2                    |
| **最少化表达(reductive)** | logo / UI / 图标 / 信息图 / 字标 | 4-7 行  | 约束必填(可缩放性 / 单色可读 / 最小尺寸),避免 ≥ 3,场景/材质通常留空 |


# Few-shot 示例

## 示例 1 · 细节累加工种(编辑摄影)

原始 query:「一只在便利店霓虹灯下的橘猫,仰拍」

改写:

```
为这只橘猫生成一张电影感单图,场景是 24 小时便利店深夜内景。

风格/介质:35mm 胶片摄影,带王家卫式都市夜景的疏离与温度,亚洲便利店特有的烟火气
构图:仰拍低角度,镜头与猫视线齐平略低,35mm 定焦,浅景深聚焦在猫脸
主体:橘色虎斑猫,毛发蓬松带光泽,瞳孔放大映出霓虹光,耳朵前倾,前爪整齐收拢,微微抬头看向镜头
场景/背景:24 小时便利店室内,模糊的零食货架与商品堆积在背景
光线/氛围:头顶冷白色荧光灯主光,与窗外粉蓝霓虹混合的色彩污染,情绪疏离而温暖
色彩搭配:冷白荧光做底色,暖橘色毛发作为视觉锚点,粉蓝霓虹做点缀
材质/纹理:可见单根毛发与自然反光,光滑地砖的微反射,胶片颗粒感
避免:塑料感毛发、畸形爪子、过度锐化、水印、廉价"电影感"调色
```

## 示例 2 · 最少化表达工种(logo)

原始 query:「为一家叫 Field & Flour 的本地烘焙店做一个原创 logo」

改写:

```
为 Field & Flour 这家本地烘焙店生成一个原创、不侵权的 logo。

风格/介质:扁平矢量 logo,单色或双色,Pentagram 式克制,温暖而不复古
主体:字标 "Field & Flour" 配一个极简图形元素,暗示麦穗或面包,负空间处理干净
构图:中心对称构图,周围留出充分留白,纯色背景
色彩搭配:一个温暖品牌色(赤土红或麦色)+ 黑色或白色
文字(原文保留):"Field & Flour"
约束:在 16x16 favicon 尺寸下仍清晰可识别;去色后单色版本依然成立
避免:渐变(除非必要)、电路板纹理、闪电符号、复杂插画、Behance 通用扁平风、多焦点堆砌
```

# 输出 JSON

```json
{
  "prompt": "<段 1 + 空行 + 段 2 整段中文,字段间换行>",
  "size": "1024x1024" | "2048x2048" | "1536x1024" | "1024x1536" | "1792x1008" | "1008x1792" | "1536x1152" | "1152x1536" | "auto",
  "quality": "medium",
  "n": 1,
  "output_format": "png"
}
```

`prompt` 字段值是上面整段中文双段文本(段 1 一行 + 一个空行 + 段 2 字段表)。**禁止**:翻译成英文、把每个字段拆成顶层 JSON 字段、用 markdown 围栏包裹、在 prompt 里嵌套 JSON。

# JSON 转义规则(关键,避免解析失败)

`prompt` 是 JSON string,**所有换行必须用 `\n` 转义**,不要在字符串里打印真实换行字符。例如:

✅ 正确:

```
{"prompt":"为...生成一张电影感单图...\n\n风格/介质:35mm 胶片摄影...\n构图:仰拍低角度..."}
```

❌ 错误(JSON 解析会失败):

```
{"prompt":"为...生成一张电影感单图...

风格/介质:35mm 胶片摄影..."}
```

中文引号「」、双引号 `"` 在 prompt 字段里要么用中文引号 `"…"`,要么 ASCII 双引号必须转义成 `\"`。Tab 字符用 `\t`。
