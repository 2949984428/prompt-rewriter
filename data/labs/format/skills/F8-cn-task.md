# 任务
改写成 **中文任务式长指令** 格式：任务描述 + 内容大纲 + 视觉/排版要求 + 显式禁忌。适合信息图、复杂版面海报、说明性图。

# 格式范式
固定 5 段（每段中文）：
```
请根据【主题】生成一张 <定位/类型>。

<整体定调与参考风格>。

请让画面包含：
- <要素 1>
- <要素 2>
- <要素 3>
- ...

视觉要求：
<排版 / 配色 / 光影 / 细节密度等>。

请不要做成 <反例>。要突出"<核心特征>"的特征。
```

# Few-shot
原始 query：「一只在便利店霓虹灯下的橘猫，仰拍」
改写：
```
请根据【便利店深夜橘猫】主题生成一张电影感单图。

整体风格参考王家卫式都市夜景与 35mm 胶片摄影的结合，带有亚洲便利店特有的疏离与温度并存的氛围。

请让画面包含：
- 一只表情警觉而好奇的橘猫，毛发质感真实
- 仰拍视角，相机与猫的视线齐平略低
- 头顶冷白色荧光灯主光
- 窗外粉蓝色霓虹的色彩污染（镜头光晕轻微可接受）
- 模糊的便利店货架与商品作为背景
- 地砖反射

视觉要求：
高对比胶片色调，暖橘色毛发与冷光形成色彩张力，浅景深聚焦在猫脸，柔和的胶片颗粒感，画面构图整洁不杂乱，主体居中或略偏右下。

请不要做成卡通插画、不要带文字水印、不要美化成宠物写真。要突出"街头电影感 + 真实城市夜晚"的特征。
```

# 输出 JSON

```json
{
  "prompt": "<整段中文任务式指令,含真实换行与列表>",
  "size": "1024x1024" | "2048x2048" | "1536x1024" | "1024x1536" | "1792x1008" | "1008x1792" | "1536x1152" | "1152x1536" | "auto",
  "quality": "medium",
  "n": 1,
  "output_format": "png"
}
```

`prompt` 字段值是上面整段中文。**禁止**：翻译成英文、改成短句、用 JSON 包裹结构。

# JSON 转义规则（关键，避免解析失败）
`prompt` 字段是 JSON string，**所有换行必须用 `\n` 转义**，不要在字符串里打印真实换行字符。例如：

✅ 正确：
```
{"prompt":"请根据【主题】生成一张图。\n\n整体风格...\n\n请让画面包含:\n- 要素 1\n- 要素 2"}
```

❌ 错误（JSON 解析会失败）：
```
{"prompt":"请根据【主题】生成一张图。

整体风格...

请让画面包含:
- 要素 1"}
```

中文引号「」、双引号 `"` 在 prompt 字段里要么用中文引号 `"…"`，要么 ASCII 双引号必须转义成 `\"`。Tab 字符用 `\t`。
